# Email Monitoring Directory

This directory contains email infrastructure monitoring data, independent of customer application code.

## Structure

```
email_monitoring/
├── dmarc_reports/     # Raw DMARC reports (XML, ZIP) from mail providers
├── reports/           # Analyzed Markdown reports with statistics
└── analysis/          # Historical analysis data (future use)
```

## Data Sources

- **DMARC Reports:** Downloaded daily via `scripts/fetch_dmarc_reports.py`
- **Analysis:** Generated by `scripts/analyze_dmarc_failures.py`

## Usage

### Fetch New Reports
```bash
cd ~/minipass_env
python scripts/fetch_dmarc_reports.py
```

### Analyze All Reports
```bash
python scripts/analyze_dmarc_failures.py
```

### View Latest Report
```bash
ls -lt email_monitoring/reports/*.md | head -1 | awk '{print $NF}' | xargs cat
```

## Automation

Daily automated fetch via cron:
```bash
0 10 * * * cd /path/to/minipass_env && source ~/.mailpass && python scripts/fetch_dmarc_reports.py >> /var/log/dmarc_fetch.log 2>&1
```

## Data Retention

- Raw reports: Keep indefinitely for audit trail
- Analysis reports: Keep 90 days (quarterly review)
- Historical data: Archive monthly

## Security

This directory is in `.gitignore` - data is not committed to git.
DMARC reports may contain sensitive information (IP addresses, email volumes).
