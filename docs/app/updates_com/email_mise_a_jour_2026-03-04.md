Un peu plus longtemps que d'habitude entre les mises à jour — la dernière remontait au 14 février. Ce délai était intentionnel : nous avons effectué un travail de fond important sur l'infrastructure courriel (livraison, conformité, surveillance) et nous voulions nous assurer que tout était stable avant d'aller de l'avant.

Cette mise à jour est la plus substantielle depuis le lancement. Beaucoup de changements en même temps signifie un risque plus élevé d'anomalies. Si tu remarques quoi que ce soit d'inhabituel, fais-le moi savoir immédiatement.

## Améliorations

- **Nouveau : flux d'inscription « Paiement d'abord » ou « Approbation d'abord »** - Tu peux maintenant choisir, pour chaque activité, comment les inscriptions fonctionnent. Soit le participant paie directement pour confirmer sa place, soit il soumet une demande qui doit être approuvée (ou refusée) avant qu'un paiement soit demandé. Ce choix est visible directement sur le tableau de bord pour chaque activité.

- **Nouveau : accepter les paiements par carte de crédit (Stripe)** - Tes participants peuvent maintenant payer par carte de crédit en ligne, directement via le formulaire d'inscription. Pour commencer, il suffit de créer un compte Stripe — nous avons préparé un guide étape par étape sur notre site pour rendre la configuration aussi simple que possible.

- **Rapport financier unifié (Interac + Stripe)** - Le rapport financier regroupe maintenant les paiements Interac et les paiements Stripe dans un seul rapport normalisé. Peu importe le mode de paiement utilisé par tes participants, tout apparaît de façon cohérente dans le même document.

- **Tableau de bord amélioré** - Quelques ajouts pratiques sur le tableau de bord : tu peux maintenant copier le lien du formulaire d'inscription directement dans le presse-papiers d'un seul clic, et le mode d'inscription de chaque activité (paiement d'abord ou approbation d'abord) est affiché clairement.

- **Nouveau : annonces par activité** - Tu peux maintenant envoyer une note rapide à tous les participants d'une activité — directement depuis la page de l'activité. Un courriel est envoyé à la liste. Si tu utilises Discord, tu peux aussi copier l'annonce dans n'importe lequel de tes canaux configurés.

- **Création de passeport depuis un paiement Interac non-apparié** - Lorsqu'un paiement Interac arrive sans correspondance automatique, le courriel de l'expéditeur est maintenant pré-rempli dans le formulaire. Il est également possible d'ajuster la quantité de sessions pour qu'elle corresponde exactement au montant reçu.

- **Amélioration des courriels** - Les courriels de confirmation ont été clarifiés. Le courriel de réinitialisation de mot de passe ne se retrouve plus dans le dossier indésirable. Le courriel de bienvenue à l'inscription a été amélioré pour être plus clair et mieux livré.

- **Optimisation des performances** - L'application se charge plus rapidement grâce à des améliorations côté serveur.

- **Correction du code QR** - Un bug affectant les codes QR a été corrigé.

- **Correction de l'affichage mobile (page de paiement)** - L'affichage sur les petits écrans a été corrigé et amélioré sur la page de paiement.
